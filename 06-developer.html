  • Bug traige:
    • track down and shoot Fedora bugs in http://bugzilla.redhat.com
    • bridge the gap between users and developers to aid in fixing bugs
  • Infrastructure:
    • help provide servers, tools, and utilities for the community
    • create new applications to make development of Fedora smoother
  • Release engineering:
    • maintain the Fedora build system
    • maintain the package repositories
    • create and release new Fedora versions
  • Websites:
    • improve Fedora's image and create better interfaces for users and contributors on the Internet
